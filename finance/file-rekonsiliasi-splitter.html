How the ACMM Sales File Splitter works

1

Upload ACMM File

Upload the Transaction Summary Details Excel export from ACMM system.

2

Split by GL Code

Each unique GL Account Code (Col M) becomes its own sheet. Original sheet is preserved.

3

Net per Kode Rekon

Kode Rekon = Day + Store Code (from brackets). Debit summed; Credit subtracted.

4

Download Output

Each GL sheet: Col A = Kode Rekon, Col B = Net Amount. Original sheet preserved.

How the BRI Statement Splitter works

1

Upload 2 Files

BRI Statement CSV (PKUTAMA_...) + BRI Master MID XLSX (MID → Outlet Code mapping).

2

Filter Cr Only

Only rows where GLSIGN = Cr (Col L) are processed. Db rows are ignored.

3

MID → Outlet + Kode Rekon

MID (001999xxxxxx) extracted from DESK_TRAN, matched to Outlet Code. Kode Rekon = Day + OutletCode (e.g. 1JKJSTT1).

4

Sum Gross Amount

AMT from description (e.g. AMT:4.127.551,00 → 4127551) summed per Kode Rekon. Output: Col A = Kode Rekon, Col B = Total Gross.
